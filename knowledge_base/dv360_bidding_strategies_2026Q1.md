# DV360 Bidding Strategies Guide (2026 Q1)

## 1. 입찰 전략 개요

### 자동 입찰 (Automated Bidding)
| 전략 | 최적화 목표 | 필요 데이터 | 권장 상황 |
|------|------------|-------------|----------|
| Maximize Conversions | 전환 수 최대화 | 최소 15-30 전환/주 | 초기 학습, 전환 볼륨 확보 |
| Target CPA | 목표 전환당 비용 | 최소 30-50 전환/주 | 효율 관리, 안정화 단계 |
| Target ROAS | 목표 광고비 대비 매출 | 전환 가치 데이터 필수 | 매출/수익 중심 |
| Maximize Clicks | 클릭 수 최대화 | 없음 | 트래픽 확보 |

### 수동 입찰 (Manual Bidding)
| 전략 | 설명 | 권장 상황 |
|------|------|----------|
| Fixed CPM | 고정 CPM 입찰 | 브랜드 캠페인, 도달 보장 |
| Viewable CPM (vCPM) | 조회 가능 노출 기준 | 조회 가능성 중시 |

## 2. Maximize Conversions (전환 최대화)

### 작동 원리
- 예산 내에서 최대한 많은 전환 확보
- 입찰가 자동 조정 (상한선 없음)
- 학습 기간: 1-2주

### 권장 설정
```
초기 학습 기간: 2주
일일 예산: 목표 CPA × 10-15회 이상
전환 창: 클릭 후 30일 / 뷰 후 1일
```

### 장점
- 빠른 학습 속도
- 전환 볼륨 확보에 유리
- 설정이 간단

### 단점
- CPA 변동성 높음
- 예산 빠르게 소진 가능
- 효율 관리 어려움

### 사용 시나리오
1. 신규 캠페인 론칭 (데이터 없음)
2. 새로운 타겟 오디언스 테스트
3. 시즌 캠페인 (빠른 볼륨 필요)

---

## 3. Target CPA (목표 전환당 비용)

### 작동 원리
- 설정한 목표 CPA에 맞춰 입찰 최적화
- 전환 확률 높은 사용자에게 높은 입찰
- 학습 기간: 2-3주

### 권장 설정
```
목표 CPA: 과거 30일 평균 CPA × 1.0-1.2
일일 예산: 목표 CPA × 10회 이상
최소 전환: 주 30-50회 (학습 유지)
```

### 목표 CPA 설정 가이드

| 과거 데이터 | 초기 목표 CPA |
|-------------|---------------|
| 있음 (안정적) | 평균 CPA × 1.0 |
| 있음 (변동) | 평균 CPA × 1.1 |
| 없음 | 예상 CPA × 1.2 |

### 주의사항
- 목표 CPA 너무 낮게 설정 시 노출 제한
- 주 30회 미만 전환 시 학습 불안정
- 큰 폭의 목표 변경 시 재학습 필요

---

## 4. Target ROAS (목표 광고비 대비 매출)

### 작동 원리
- 설정한 ROAS 목표에 맞춰 입찰 최적화
- 전환 가치가 높은 사용자에게 높은 입찰
- 학습 기간: 2-4주

### 필수 조건
- Floodlight에 전환 가치(Revenue) 전달 필수
- 최소 15-20회/주 전환 (가치 포함)

### 권장 설정
```
목표 ROAS: 과거 30일 평균 ROAS × 0.9-1.0
전환 가치 전달: Floodlight → Revenue 파라미터
```

### ROAS 계산
```
ROAS = 전환 매출 / 광고비
예: 1,000,000원 매출 / 300,000원 광고비 = 3.33x ROAS
```

### 업종별 ROAS 벤치마크 (한국)

| 업종 | 평균 ROAS | 상위 25% |
|------|----------|----------|
| 건강기능식품 | 3.5x | 5.0x+ |
| 패션/의류 | 4.0x | 6.0x+ |
| 전자제품 | 2.5x | 4.0x+ |
| 금융 서비스 | 5.0x | 8.0x+ |
| 교육 | 3.0x | 4.5x+ |

## 5. Fixed CPM (고정 CPM)

### 작동 원리
- 설정한 CPM으로 고정 입찰
- 노출 보장형 캠페인에 적합
- 학습 기간: 없음

### 권장 설정
```
입찰 CPM: 시장 평균 CPM × 1.1-1.5
(높은 입찰 = 더 많은 인벤토리 접근)
```

### 한국 시장 CPM 참고값

| 인벤토리 유형 | 평균 CPM |
|--------------|----------|
| Open Exchange (Display) | ₩1,500-3,000 |
| Open Exchange (Video) | ₩5,000-12,000 |
| PMP (Display) | ₩3,000-8,000 |
| PMP (Video) | ₩10,000-25,000 |
| Programmatic Guaranteed | ₩15,000-40,000 |

## 6. 입찰 전략 전환 가이드

### Phase 1: 론칭 (1-2주)
```
전략: Maximize Conversions
목적: 초기 데이터 수집, 알고리즘 학습
예산: 충분히 설정 (일일 목표 CPA × 10 이상)
```

### Phase 2: 안정화 (2-4주)
```
전략: Target CPA
목적: 효율 관리, CPA 안정화
목표: 과거 2주 평균 CPA × 1.0
```

### Phase 3: 최적화 (4주+)
```
전략: Target ROAS (전환 가치 있는 경우)
목적: 수익 최적화
목표: 비즈니스 목표에 맞는 ROAS
```

### 전환 시점 체크리스트
- [ ] 주간 전환 수 30회 이상 달성
- [ ] CPA 변동성 20% 이내로 안정
- [ ] 예산 소진율 80% 이상 유지
- [ ] Floodlight 데이터 정상 수집

## 7. 입찰 조정 (Bid Adjustments)

### 디바이스별 조정
| 디바이스 | 권장 조정 | 상황 |
|----------|----------|------|
| Mobile | +20% | 모바일 전환율 높은 경우 |
| Desktop | -10% | 데스크톱 성과 낮은 경우 |

### 시간대별 조정
| 시간대 | 권장 조정 |
|--------|----------|
| 피크 타임 (18-22시) | +15-25% |
| 점심 시간 (12-14시) | +10-15% |
| 심야 (00-06시) | -20-30% |

### 지역별 조정
- 고성과 지역: +10-20%
- 저성과 지역: -10-20%
- 성과 데이터 없는 지역: 기본 유지

## 8. 트러블슈팅

### 문제: 예산 미소진
| 원인 | 해결 방법 |
|------|----------|
| 목표 CPA 너무 낮음 | CPA 10-20% 상향 |
| 타겟팅 너무 좁음 | 오디언스 확장 |
| 입찰가 너무 낮음 | CPM 상향 조정 |

### 문제: CPA 목표 초과
| 원인 | 해결 방법 |
|------|----------|
| 경쟁 심화 | 시간대/지역 조정 |
| 타겟 품질 낮음 | 오디언스 재검토 |
| 크리에이티브 피로 | 소재 교체 |

### 문제: 전환 수 급감
| 원인 | 해결 방법 |
|------|----------|
| 시즌 종료 | 메시지/타겟 변경 |
| 알고리즘 재학습 | 2주 대기 |
| Floodlight 오류 | 태그 점검 |
